区域

参数请求说明返回值
uid get 发送用户uid，将返回用户UID编号值的所有区域信息,其中pretermit1代表是默认区域设备，不能删除 {code: 200, msg: "获取区域分类成功", data: [{rid: "2", name: "客厅", pretermit: "1"}, {rid: "3", name: "厨房", pretermit: "1"},…]}
uid,name post UID用于判定身份，name为区域名称 {code: 200, msg: "区域添加成功", data: []}
uid,rid,name put 用户名，区域RID，区域名称，修改该区域的名称 {code: 200, msg: "区域修改成功", data: []}
uid,rid delete 用户uid和区域RID 。用于删除该区域 {code: 200, msg: "区域删除成功", data: []}

示例：

获取情景：发送uid用GET请求域名/版本/region/index/1

返回值：{code: 200, msg: "获取区域分类成功", data: [{rid: "1", name: "客厅"}, {rid: "3", name: "书房"}]}

添加情景：用户发送uid以及添加的情景名称name POST请求到域名/版本/region/index/1

返回值：{code: 200, msg: "区域添加成功", data: []}

修改情景：发送用户uid和json数组rid,name。如{"rid": "3", "name": "厨房"}put到域名/版本/region/index/1

返回值：{code: 200, msg: "区域修改成功", data: []}

删除情景：发送用户uid,情景rid的json数值如{"rid": "3"} DELETE请求到域名/版本/region/1

返回值：{code: 200, msg: "区域删除成功", data: []}
